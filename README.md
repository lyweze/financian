# Financian

Приложение на React + TypeScript для учета финансов: доходы, расходы, цели, подсказки, аналитика и визуализация. Подходит для личного или семейного бюджета, демонстрирует базовые подходы к работе с состоянием, компонентами и данными в SPA.

## Возможности
- Добавление доходов и расходов через модальные окна с валидацией.
- Последние операции, цели, советы и напоминания.
- Графики и сводки по категориям, тегам и месяцам.
- Навигация по ключевым разделам (главная, аналитика, цели).
- Настраиваемые категории, суммы, теги, комментарии.
- Аналитический блок с ключевыми метриками и подсветкой трендов.
- Локальное сохранение данных (стор, localStorage при необходимости).

## Технологии
- React 18 + TypeScript
- Vite
- CSS Modules (локальные стили компонентов)
- Zustand (управление состоянием)
- ECharts (или альтернативный графический пакет)
- ESLint + Prettier (код-стайл)

## Установка и запуск
```bash
# Установка зависимостей
npm install

# Запуск dev-сервера
npm run dev

# Сборка проекта
npm run build

# Предпросмотр собранной версии
npm run preview
```

### Требования
- Node.js LTS
- npm или pnpm/yarn
- Поддержка современных браузеров

## Структура проекта
```
src/
    components/
        add-income-modal/    # Модалка добавления дохода
        add-waste-modal/     # Модалка добавления расхода
        header/              # Шапка и эффект Liquid
        main/                # Главная: кнопки, категории, графики, цели, операции, советы
        navigation/          # Навигационное меню
    store/                 # Store.ts, useStoreSnapshot.ts
    utils/                 # Вспомогательные функции, tips.ts
    styles/                # Глобальные стили/переменные (если есть)
```

## Скрипты npm
- `npm run dev` — режим разработки.
- `npm run build` — сборка.
- `npm run preview` — предпросмотр сборки.
- `npm run lint` — проверка кода (если настроено).

## Настройка
- Проверьте `.env` для переменных окружения (API, базовые URL и т.п.).
- Обновите категории и теги под свой сценарий.
- При интеграции с API настройте запросы в `utils` или сторе.

## Разработка и вклад
- Форкните репозиторий, создайте ветку feature/...
- Соблюдайте форматирование ESLint/Prettier.
- Добавьте тесты/сторибуки для новых компонентов (по возможности).

## План развития
- Импорт/экспорт данных (CSV/JSON).
- Авторизация и синхронизация с бэкендом.
- Более детальная аналитика и уведомления.
